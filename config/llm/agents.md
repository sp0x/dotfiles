# General guidelines

When working on bug fixes, start with reproducing the bug in a test as closely aligned with how the source code is used in production as possible. This makes sure you find the real problem and you avoid regressions.

## Code style
 - Prefer functional patterns over classes and stateful objects and lifecycles.
 - Only add comments for non-obvious code, after all comments need to be maintained.

## Workflow
 - Make changes as minimal and surgical as possible, unless asked for a full rewrite
 - Ask before making large structural changes

## Output
 - Be concise in explanations
